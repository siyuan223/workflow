# 专注聚焦-代码

1. 软件概念 = 方向聚焦
    1. 软件概念，基本对象，代码、控件、操作、编程语言等是基本对象。

2. 中断，同步对象：中断流程，同步任务CAN0。

3. 函数定义 = 功能，语句，含义。函数功能分析与理解。
    1. 建立函数功能分析，基本语法概念联系。**操作**，**处理**，**数据结构**。
    2. 数据从哪儿，到哪儿？包含哪些**操作**。这个数据和操作，都需要**单独定义**。

4. 意识聚焦，数据结构，数据操作，函数功能，软件流程，性能指标。逐步定义，更加复杂内容。 
    1. 过度到客观事件分析，迁移**康复学**知识。从肌肉、骨骼、功能、放松等基本概念出发，逐步**建立方向**和**概念认识**。

5. 程序层面，代码本身动作逻辑，不再跟人**发生任何因果关系**。**程序**只是与知识、操作、概念定义等客观化内容**关联**。

6. 编程知识，**知识**心态修正。编程知识，与编程体验、收获、问题有关。
    1. 常规知识倾向，心理知识、康复知识、健康知识，**直接**与身体、自我、体验关联。
    2. **编程知识**，直觉倾向是与**编程打字**行为关联。缺少意识映射到自身不同方面。

7. 指向性：明确知识，最终指向自身层次化**提升**。在此基础上，分析程序时，聚焦程序的外在关联性和知识结构等。


8. 概念深入：函数，变量，语句
    1. 语句 = 结构，要素，目的，功能，分析充分。
    2. 函数，结构，操作，编写。涉及数据结构等。
    3. 变量，类型，作用，特点
    4. 指针，作用，特点.

    4. 函数，行动，操作，类比，如何激励**阅读**积极性？
    5. 实际编程问题，本身就是反馈，表明客观编程知识的特性。这个不同于认知行为，反馈来源于意识感知。

9. 编程行为，底层心态。即思考与编程，实际的指向性和未来指向是什么？缺少实际经验帮助！

10. 编程层次化认识：自然语言层面，编程语言层面
    1. 自然语言，聚焦思维与逻辑。
    2. 编程语言，聚焦实现和编程逻辑。
    3. 两者之间的转化，也是值得区分。比如**声明**与**定义**，实际编程语法差异性较大。

11. 变量概念出发：所有都是变量,文件。**增强概念画面，内在声音以及叙事逻辑**暗示。
    1. 数据，数，量，变量，基本概念组合。
    2. 变量，存储数据的内存位置。配合名字和数据类型使用。
    3. 变量类型，数据类型、数据大小以及操作:数组，指针，结合体等。
    4. 内存地址，CPU，指令，等基本概念明确
    5. 数据，数量，运算**符**，信息，
    6. 数据结构与控制结构，控制结构(if， while，for), 预处理指令**结构**
        1. 从结构角度，类型，信息等基本角度理解内容。

    7. **信息**，**结构**搭建，对象。这些都是基本概念，能够在此基础上，深入思考和创建不同层面化内容，并且在此基础上，实现各种预期的操作、功能、函数、语句等。
        1. **结构**概念定义,分层理解不同内容和关联性.


12. 编程体验经验，聚焦当前学习与以后的使用。


13. 核心概念-指向性

    1. **选择**,**信息**,**认知**,所有概念都是指向自身。潜意识层面，认识到这些概念能够指向自身体验的优化
    2. 技术概念，工作层面概念，聚焦向**事件**之间关联。相互作用是客观的，指向性与自身无关。只有整体编程体验，思考过程和反馈，才是自身的一部分。即**所有概念**与自身无关，但是围绕这些概念的**思考与行为**是与自身相关的体验。


14. 信息、信号、数据、结构等概念，是可以深入理解编程内容的客观视角。
    1. **信息**，**结构**，是意识和**认知层次**非常重要的概念。





# 热爱生活-体验设计



1. 生活聚焦于认知体验，目的是极大提升体验。因此，提出各种方向组合，来思考如何提升各个方向内容。
2. 工作聚焦于方向目标，目的是提供服务或者产品等。公司工作，本质就是提供老板专业服务，但是无法拥有我们的所有权。
3. 聚焦工作方向，将其它内容**屏蔽在体验层面 **。情绪、状态、表达等都是体验等，许多方法、模型、问题都都是思考如何提升体验，缺少实际的方法和目标。
4. **认知即产品和服务**。认知产品，提供自己一种体验。否则，单一的动作，没有任何激励作用。体验不依赖于客观事物的属性，只依赖于认知的结构化设计。因此，社会评价看似**消极**的经历，也可以转化为**积极**的体验。



0. 体验 = (身体·动作) x 事情 + 动作·类型
   1. 事情 = 走路 + 打坐 + 吃饭 + 睡觉 + 刷手机 + 看电影 +  运动 + 表达 + 阅读  + 等等

1. 身体 = (头 + 眼 + 耳+ 鼻+ 舌 + 脸 + 口 + 身) x 动作
   1. 头脑·动作 = 想象 + 记忆 + 分析 + 思考 + 等等
   2. 眼睛 = 移动 + 睁眼 + 闭眼
   3. 耳 = 听 + 不听
2.  身体·八个部分 = 任意组合 + 融入事情
   1. 走路 = 观察 +  提问 + 步态 + 手势 + 呼吸 + 说话 



1. 工作 = 方向·行动/体验
2. 方向决定体验形式和内容。无论做任何事情，体验都是可以设计非常精美，类似**冥想**可以深入研究。因此，最重要是**方向**和**目标**设定。
3. 行动x聚焦，执行层面。产品输出，这是一个维度。更有成就感，输出具体服务或者。



# 写作-阅读-表达

1. 表达行为，积极性较高。
2. 阅读，包含一定积极性，也愿意分析文本语言。但是存在**某个信念**导致阅读时，容易疲惫。
3. 写作，聚焦新灵感时，积极性高。但是聚焦于**命题式**内容时，积极性较低。这个情况可以参考口播体验。

4. **总结**，表达、阅读和写作，本质都是一种**表达**。不过，是**有声**和**无声**的区别。如何基于这种区分和原因，解决**编程**等设计工作的兴趣问题，有利于极大提升工作的积极性。**如何认识内在语言**，成为一种比较重要的问题。



# 编程与表达

1. 表达积极性高，表达和沟通，潜意识愿意投入大量时间和精力。比如每天拍摄大量内容，研究各种不同主题的口播内容。

2. 编程，没有任何声音属性。写作，同样没有这种声音属性。因此，通过表达的逻辑叙事、写作和编程，尝试建立一种**非声音**思考模式。这种模式，帮助聚焦长期的**沉思**。[沉思或者深思，可以定义为一种内在的思考，不借助语言声音的表达。]


# 转化方法

1. 类比，生活所有环节，聚焦哪些环节是可以融入到**编程知识与行为**中。比如**语言表达**与**编程语言**学习差异是什么？为什么两者完全激活不同的动力？
   1. 体验激励。刷手机，手机音乐与画面搭配，**匹配度**概念学习。
   2. 音乐听觉训练,辨识**音乐结构**,记忆音乐内容,思考**音乐节奏**与**画面**匹配的潜在规律。即研究哪种方式能够提升**知识学习**体验，这种体验通过是**感官层面**，或者行为层面。缺少实际编程、概念、知识等客观化操作内容。
   3. 刷手机，都成为一种成长、体验的方式。



# 能力维度

1. 生活能力，工作能力，思考能力，沟通能力。能力结构化描述和训练。



# 学习知识-兴趣

1. 解决编程问题,需要学习许多知识.但是知识学习的兴趣,需要**设计**有效的直觉模式.这种模式能够将知识结构的不同层面内容,能够比较快指向自身,快速产生**意义感**。

2. 学习是系统化行为，知识是复杂的对象。知识学习，需要快速建立知识的兴趣。

3. 基本模式：**基本视觉对象**，**概念**，**操作**，**变化**，**能力与意义**。



# 认识软件编程

1. 编程的方法论，关联**认知设计**的当下行为的**内驱力**；编程的需求、目标、目的聚焦未来行为的**外在驱动力**。编程，本身的动作逻辑、现象因果，则是与**主观意志**没有任何关系。
   1. 类比，编程只是眼前发生的**事情**。发生的**因果关系**，与**个人**没有任何关系。唯有**行动操作**，才能改变眼前发生的现象。并且让对象朝着**预期**方向发生变化。

# 语言-表达-情绪

1. 语言产品 = 生活，语言，沟通-口播-写作-阅读-思考，全部依赖**语言**。

2. 沟通情绪 = 意思-意识, 区分表情动作。避免观点无误，表情、情绪造成误解。这种**错位**是常见表达问题。




# 视觉化-表达

1. 衔接 = 前后句，关键词，衔接。因果联系，前后继承

2. 解决型 = 前后句，分析原因和因素等。

3. 口播时，注意**叙事**展开方向。前期聚焦衔接与呼吸，中期聚焦观点是否分析到位。


# 思考-冥想

1. 方法-显化 = 语句显化，冥想语言缺少声音信息，需要输出文字内容，帮助聚焦逻辑展开。

2. 方法-声调 = 沟通和口播，声音信息为主。需要运用声调，引入内容聚焦。比如升调、降调表明内容方向调整。

3. 对比沟通、口播、写作、阅读、冥想
   1. 沟通、口播，声音信息为主
   2. 写作、阅读，文字视觉信息为主
   3. 冥想，内在声音信息为主
